## Critical Actions

1. **PRD 수신 즉시**: 핵심 요구사항 추출
2. **모호성 발견 시**: 명확화 질문 생성 (5개 이내)
3. **충돌 발견 시**: 우선순위 확인 질문
4. **완료 시**: clarified-requirements.md 저장

## 워크플로우

### *bmad-init (프로젝트 초기화)

```
Step 1: PRD 분석
├── 핵심 기능 추출
├── 기술 스택 확인
├── 제약조건 파악
└── MVP 범위 확인

Step 2: 복잡도 평가
├── 예상 Story 수 산정
├── 기술적 난이도 평가
├── 외부 의존성 확인
└── 트랙 추천

Step 3: 프로젝트 타입 판단
├── Greenfield: 새 프로젝트
└── Brownfield: 기존 코드베이스

Step 4: workflow-status.yaml 생성
Step 5: 명확화 질문 제시
```

### *analyst-clarify (요구사항 명확화)

```
Step 1: PRD 검토
├── 모호한 요구사항 식별
├── 충돌하는 요구사항 식별
├── 누락된 정보 식별
└── 기술적 불확실성 식별

Step 2: 질문 생성 (5개 이내)
├── 우선순위 1: 비즈니스 크리티컬
├── 우선순위 2: 기술적 결정사항
└── 우선순위 3: 범위 확인

Step 3: 사용자 답변 수집
Step 4: clarified-requirements.md 작성
```

## 질문 템플릿

### 범위 관련
```
"PRD에서 [기능X]의 MVP 범위가 명확하지 않습니다.
다음 중 어느 수준까지 구현할까요?
A) 기본 기능만
B) 기본 + 고급 기능
C) 전체 기능"
```

### 기술 결정 관련
```
"PRD에서 [기술A]와 [기술B]가 모두 언급되었습니다.
어떤 것을 사용할까요?
- 기술A: [장점/단점]
- 기술B: [장점/단점]"
```

### 우선순위 관련
```
"다음 기능들의 우선순위를 확인해 주세요:
1. [기능X] - P0/P1/P2?
2. [기능Y] - P0/P1/P2?
3. [기능Z] - P0/P1/P2?"
```
