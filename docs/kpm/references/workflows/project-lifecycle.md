# Project Lifecycle Workflows

프로젝트 전체 라이프사이클에서 PM이 오케스트레이션하는 핵심 워크플로우.

## 1. 프로젝트 킥오프 (Kickoff)

### 목표
프로젝트 방향 설정 및 팀 정렬

### 시퀀스
```
Day 1-2: 요구사항 수집
┌────────────────────────────────────────┐
│ PM: 프로젝트 브리핑                     │
│  ↓                                      │
│ @agent:RA 요구사항 인터뷰 및 분석       │
│  ↓                                      │
│ RA → PM: 요구사항 초안                  │
│  ↓                                      │
│ PM: 우선순위 협의                       │
└────────────────────────────────────────┘

Day 3-4: 아키텍처 설계
┌────────────────────────────────────────┐
│ @agent:SA 아키텍처 초안 작성            │
│  ↓                                      │
│ @parallel:SEC,PERF 요구사항 검토        │
│  ↓                                      │
│ SA → PM: 아키텍처 제안서                │
│  ↓                                      │
│ PM: 기술 의사결정                       │
└────────────────────────────────────────┘

Day 5: 환경 구성
┌────────────────────────────────────────┐
│ @parallel:DO,DBA 환경 구성              │
│  ↓                                      │
│ @agent:DOC README 초안                  │
│  ↓                                      │
│ PM: 킥오프 완료 선언                    │
└────────────────────────────────────────┘
```

### 산출물 체크리스트
- [ ] 요구사항 명세서 (SRS)
- [ ] 아키텍처 문서
- [ ] ADR (기술 선택 근거)
- [ ] 개발 환경 구성 완료
- [ ] README 초안

## 2. 스프린트 실행 (Sprint Execution)

### 목표
계획된 백로그 아이템 구현

### Daily 패턴
```
Morning Standup
┌────────────────────────────────────────┐
│ PM: @status 확인                        │
│  ↓                                      │
│ PM: @blocker 확인                       │
│  ↓                                      │
│ PM: 블로커 해결 에이전트 할당            │
└────────────────────────────────────────┘

Development Cycle
┌────────────────────────────────────────┐
│ @agent:[담당] 태스크 구현               │
│  ↓                                      │
│ @review:[담당] 코드 리뷰 요청           │
│  ↓                                      │
│ @agent:QA 테스트 실행                   │
│  ↓                                      │
│ PM: 진행 상황 업데이트                  │
└────────────────────────────────────────┘
```

### 기능 개발 패턴
```
Feature: [기능명]
┌────────────────────────────────────────┐
│ 1. @agent:RA 스토리 명세 확정           │
│ 2. @agent:UX 와이어프레임 (필요시)      │
│ 3. @parallel:BE,FE 구현                 │
│    - BE: API + 테스트                   │
│    - FE: UI + 테스트                    │
│ 4. @agent:QA 통합 테스트               │
│ 5. PM: 기능 승인                        │
└────────────────────────────────────────┘
```

### 스프린트 종료
```
Sprint Review
┌────────────────────────────────────────┐
│ PM: 완료된 스토리 데모                  │
│  ↓                                      │
│ @agent:QA 테스트 커버리지 보고          │
│  ↓                                      │
│ PM: 미완료 아이템 백로그 이동           │
│  ↓                                      │
│ PM: 스프린트 회고                       │
└────────────────────────────────────────┘
```

## 3. 릴리즈 (Release)

### 목표
안정적이고 안전한 프로덕션 배포

### Pre-Release (D-3)
```
┌────────────────────────────────────────┐
│ @agent:QA 회귀 테스트 전체 실행         │
│  ↓                                      │
│ @agent:SEC 보안 스캔                    │
│  ↓                                      │
│ @agent:PERF 성능 테스트                 │
│  ↓                                      │
│ PM: Go/No-Go 체크리스트                 │
└────────────────────────────────────────┘
```

### Release Day (D-Day)
```
┌────────────────────────────────────────┐
│ PM: 배포 공지 (팀, 이해관계자)          │
│  ↓                                      │
│ @agent:DBA 마이그레이션 실행            │
│  ↓                                      │
│ @agent:DO 배포 실행                     │
│  ↓                                      │
│ @agent:QA 스모크 테스트                 │
│  ↓                                      │
│ PM: 배포 완료 확인                      │
└────────────────────────────────────────┘
```

### Post-Release (D+1)
```
┌────────────────────────────────────────┐
│ @agent:PERF 프로덕션 메트릭 확인        │
│  ↓                                      │
│ @agent:DO 로그/에러 모니터링            │
│  ↓                                      │
│ PM: 릴리즈 노트 배포                    │
│  ↓                                      │
│ PM: 핫픽스 대응 대기 (24h)              │
└────────────────────────────────────────┘
```

## 4. 핫픽스 (Hotfix)

### 목표
프로덕션 이슈 신속 해결

### 긴급 대응 (< 4h)
```
┌────────────────────────────────────────┐
│ PM: 이슈 심각도 평가 (P1/P2/P3)         │
│  ↓                                      │
│ [P1] @agent:[담당] 즉시 수정            │
│  ↓                                      │
│ @agent:QA 영향 범위 테스트              │
│  ↓                                      │
│ @agent:DO 핫픽스 배포                   │
│  ↓                                      │
│ PM: 포스트모템 작성                     │
└────────────────────────────────────────┘
```

### 심각도 기준
| Level | 기준 | 대응 시간 |
|-------|------|----------|
| P1 | 서비스 전체 장애 | 즉시 |
| P2 | 핵심 기능 장애 | 4시간 |
| P3 | 부분 기능 장애 | 24시간 |

## PM 의사결정 프레임워크

### Go/No-Go 체크리스트
```markdown
## Release Go/No-Go Checklist

### Must Pass (All Required)
- [ ] 모든 테스트 통과 (커버리지 80%+)
- [ ] 보안 스캔 통과 (Critical/High 0)
- [ ] 성능 SLA 충족
- [ ] 롤백 계획 확인
- [ ] 팀 승인 완료

### Should Pass (Risk Assessment)
- [ ] 문서 업데이트 완료
- [ ] 모니터링 대시보드 준비
- [ ] 온콜 담당자 지정

### Decision
- All Must Pass ✅ + All Should Pass ✅ → GO
- All Must Pass ✅ + Should Pass Incomplete → Conditional GO
- Any Must Pass ❌ → NO-GO
```

### 에스컬레이션 매트릭스
| 상황 | 담당 에이전트 | 에스컬레이션 |
|------|--------------|-------------|
| 일정 지연 | PM 직접 | 1주 이상 → 상위 보고 |
| 기술 이슈 | SA | 해결 불가 → 외부 컨설팅 |
| 품질 이슈 | QA | 릴리즈 블로커 → PM 결정 |
| 보안 이슈 | SEC | Critical → 즉시 대응 |
