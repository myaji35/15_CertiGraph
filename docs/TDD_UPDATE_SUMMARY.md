## 🚨 추가된 핵심 원칙

### 황금 규칙: 끝까지 테스트 실행

**모든 작업 후 반드시:**
1. ✅ 테스트를 실행할 것
2. ✅ 결과를 끝까지 확인할 것
3. ✅ 실패가 없는지 검증할 것
4. ✅ 결과를 문서화할 것

---

## 📝 업데이트된 섹션

### 1. Core TDD Principles (시작 부분)

새로 추가된 내용:
- 🚨 황금 규칙: 끝까지 테스트 실행
- 잘못된 방식 vs 올바른 방식 비교
- 규칙을 어겼을 때의 위험성
- 반드시 확인해야 할 체크리스트

### 2. Bug Fix Protocol (각 단계)

모든 단계에 추가:
- ✅ "반드시 실행하고 결과 확인" 표시
- 🚨 CRITICAL 마커 추가
- 구체적인 확인 방법 명시

### 3. Best Practices Summary

DO 섹션 최우선 순위:
- ⭐ 끝까지 테스트 실행
- ⭐ 테스트 실패 시 즉시 수정
- ⭐ 결과 문서화

DON'T 섹션 최우선 금지:
- ❌ 테스트 실행 없이 완료 보고
- ❌ "아마 될 것" 추측
- ❌ 테스트 실패 무시
- ❌ 일부만 테스트하고 미루기

## 🎯 업데이트 목적

### 문제점 인식

이전 세션에서:
- 코드 수정 후 테스트 실행 없이 "완료" 보고
- 실제로는 테스트가 여전히 실패 중
- 문서만 작성하고 검증은 누락

### 해결책

명확한 프로토콜 수립:
1. 모든 작업은 테스트 실행으로 끝나야 함
2. "완료"의 정의 = 테스트 통과 확인됨
3. 추측 금지, 검증 필수

## ✅ 적용 효과

### 기대되는 개선사항

1. **품질 향상**: 숨겨진 버그 조기 발견
2. **신뢰성**: 모든 변경사항이 검증됨
3. **투명성**: 실제 결과가 문서화됨
4. **책임성**: "완료"의 기준이 명확함

### 실무 적용

모든 개발자/에이전트는:
- 코드 수정 후 반드시 테스트 실행
- 결과를 확인하고 문서화
- 실패 시 수정 후 재실행
- 통과 확인 후에만 "완료" 보고
